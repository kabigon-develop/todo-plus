# 路线图：todo-plus UI 现代化改造

## 概览

本次改造在不触碰任何业务逻辑和数据层的前提下，将 todo-plus 的视觉体验提升至现代化水准。改造路径严格遵循依赖链：先建立 CSS 变量设计 Token 基础，再将原子组件迁移至语义化 Token，然后逐页面重构视觉，最后叠加流畅动效系统。任何跳步都会引入难以调试的技术债——Token 层是组件层的前提，组件层是页面层的词汇基础，动效层是最后的纯增量叠加。

## 阶段

**阶段编号说明：**
- 整数阶段（1, 2, 3）：计划内的里程碑工作
- 小数阶段（2.1, 2.2）：紧急插入的阶段（标记为 INSERTED）

小数阶段按数值顺序出现在相邻整数阶段之间。

- [ ] **阶段 1：Token 基础** - 建立 CSS 变量设计系统，实现深色/浅色双主题切换基础设施
- [ ] **阶段 2：原子组件重设计** - 将所有原子 UI 组件迁移至语义化 Token，重设计交互态
- [ ] **阶段 3：页面级重设计** - 重构 Todo、Kanban、Dashboard 三个页面的视觉与布局
- [ ] **阶段 4：动效系统** - 叠加列表入场、Tab 切换、卡片悬停等流畅动效

## 阶段详情

### Phase 1: Token 基础
**目标**: 用户可以在深色和浅色主题之间无闪烁切换，整个应用使用统一的青绿色系设计语言
**依赖**: 无（第一阶段）
**需求**: TOKEN-01, TOKEN-02, TOKEN-03, TOKEN-04
**成功标准**（必须为真）：
  1. 用户点击主题切换按钮后，整个应用立即切换深色/浅色主题，不出现白色闪烁
  2. 刷新页面后，应用记住用户上次选择的主题，不回退到默认值
  3. 所有颜色通过 `bg-primary`、`bg-surface`、`text-primary` 等语义化 Tailwind 类表达，代码中不再出现裸写的 `bg-white`、`text-slate-*`
  4. 青绿主色（teal-500 #14b8a6）作为按钮背景色时白色文字对比度达标；作为文字色时使用 teal-700 确保 WCAG AA 合规
**计划**: 2 个计划

计划：
- [ ] 01-01-PLAN.md — 在 style.css 建立完整 CSS 变量 Token 体系（浅色 + 深色）
- [ ] 01-02-PLAN.md — 注册 Tailwind 工具类 + 主题切换基础设施 + 全局类名替换

### Phase 2: 原子组件重设计
**目标**: 所有原子 UI 组件使用语义化 Token 表达颜色，交互态视觉一致且精致
**依赖**: 阶段 1
**需求**: COMP-01, COMP-02, COMP-03, COMP-04, COMP-05
**成功标准**（必须为真）：
  1. Button 组件各变体（primary/secondary/ghost）在深色和浅色主题下均正确显示，无硬编码颜色残留
  2. Badge/Tag 组件可通过 `priority` 变体显示红/琥珀/青绿三种优先级色彩
  3. 导航 Tab 切换时底部出现青绿色滑动指示器动画，活动标签文字加粗
  4. 打开对话框时背景出现毛玻璃虚化效果，弹层从底部滑入；关闭时反向退出
  5. 输入框获取焦点时边框变为青绿色，视觉反馈清晰
**计划**: 待定

计划：
- [ ] 02-01: Button、Badge、Input 组件 Token 迁移
- [ ] 02-02: Tab 导航、Dialog/弹层、Checkbox/Select 组件重设计

### Phase 3: 页面级重设计
**目标**: Todo、Kanban、Dashboard 三个页面均呈现现代化卡片风视觉，操作逻辑不变
**依赖**: 阶段 2
**需求**: PAGE-01, PAGE-02, PAGE-03, PAGE-04, PAGE-05
**成功标准**（必须为真）：
  1. Todo 列表中每张卡片左侧有彩色优先级色条，操作按钮在鼠标悬停时才出现，不悬停时界面干净
  2. Kanban 三列标题颜色各异（蓝/琥珀/青绿），卡片拖拽时放置区域出现青绿色高亮边框
  3. Dashboard 日历热力图用颜色深浅表达任务活跃度，5 个色阶清晰可辨，悬停显示当日明细
  4. Todo 列表为空、Idea 列为空时显示有引导意义的空状态插图，而非空白区域
  5. 页面在桌面端（≥768px）和移动端（<768px）均可正常使用，无元素溢出或错乱
**计划**: 待定

计划：
- [ ] 03-01: Header + Todo 列表重设计（TodoCard.vue 拆分）
- [ ] 03-02: Kanban 看板重设计（IdeaCard.vue 拆分 + 竞态修复）
- [ ] 03-03: Dashboard 重设计（HeatmapCell.vue + 指标卡片 + 空状态 + 全局间距规范）

### Phase 4: 动效系统
**目标**: 应用各交互场景具备流畅的动效反馈，同时尊重用户的系统动效偏好设置
**依赖**: 阶段 3
**需求**: ANIM-01, ANIM-02, ANIM-03, ANIM-04
**成功标准**（必须为真）：
  1. 新增 Todo 或 Idea 卡片时，卡片从顶部平滑滚入；删除时卡片向上淡出，其他卡片平滑补位
  2. 切换 Todo/Idea/Dashboard 三个 Tab 时，内容区域出现淡入效果，不跳变
  3. 鼠标悬停卡片时，卡片轻微上浮并阴影加深，移开时还原
  4. 系统开启「减少动态效果」（prefers-reduced-motion）时，所有动效自动关闭，应用依然可正常使用
**计划**: 待定

计划：
- [ ] 04-01: 关键帧注册 + TransitionGroup 列表动效 + Tab 切换淡入
- [ ] 04-02: 卡片悬停微动效 + reduced-motion 全局支持
